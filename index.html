How a computer actually works, what a "program" is, and how to plan one out on paper before writing a single line of code.

1.1

Basics of Computers

1.1.1 Overview of Computer Systems

A computer is an electronic device that processes data according to a set of instructions (a program) to produce information. Every computer system rests on four basic components:

  • Hardware — the physical components of the computer
  • Software — programs and instructions that tell hardware what to do
  • Data — raw facts and figures
  • Information — processed data that has meaning

1.1.2 Hardware Components

ComponentFunction
CPU (Central Processing Unit)The brain of the computer — executes instructions
Memory (RAM)Temporary storage for data and instructions
Storage (HDD/SSD)Permanent storage for data and programs
Input DevicesKeyboard, mouse, scanner — bring data in
Output DevicesMonitor, printer, speakers — send information out
MotherboardMain circuit board connecting all components

1.1.3 Software Types

TypeDescriptionExamples
System SoftwareManages hardware and provides a platformOS, drivers, firmware
Application SoftwarePerforms specific tasks for usersWord processors, browsers, games
Utility SoftwarePerforms maintenance tasksAntivirus, disk cleanup, backup

1.1.4 Block Diagram of a Computer

Data flows through a computer in one steady direction — in, through processing, and back out:

Input UnitKeyboard, mouse, scanner
Memory UnitRAM (primary) · HDD/SSD (secondary)
CPU UnitALU + Control Unit + Memory
Output UnitMonitor, printer, speakers
  • Input Unit — converts data into computer-readable (binary) format and loads it into memory
  • Memory Unit — stores data and instructions, temporarily or permanently
  • CPU — the ALU performs arithmetic/logical operations, the Control Unit coordinates everything, and its own memory holds working data
  • Output Unit — converts processed data back into a human-readable form

A program is a set of instructions written in a programming language to perform a specific task on a computer.

1.3

Flowcharts and Algorithms

1.3.1 Algorithm

An algorithm is a step-by-step procedure to solve a problem. Five qualities make a true algorithm:

  • Input — zero or more inputs
  • Output — at least one output
  • Definiteness — each step is clear and precise
  • Finiteness — it terminates after a finite number of steps
  • Effectiveness — each step is basic and executable

1.3.2 Flowchart

A flowchart is a graphical representation of an algorithm, built from a small set of standard symbols:

Structure of a C Program

2.1.1 Basic Structure

Every C program follows the same skeleton, nested inside main():

2. Function Definitionsmain(), user-defined functions
3. Main FunctionEntry point of the program

2.1.2 Basic C Program Template

C
#include <stdio.h>      // Preprocessor directive
#include <conio.h>      // For console input/output

// Global variable declaration (optional)

int main() {            // Main function
    // Variable declaration
    int a = 10, b = 20, sum;
    
    // Execution statements

    printf("Sum = %d\n", sum);

2.1.3 Explanation of Each Section

  • Function Definitions — a C program is a collection of functions, and main() is the entry point
  • Declaration Section — where variables are declared, with their type and name
  • Execution Section — the statements that actually perform the task, using those variables and calling functions

Recursion is a technique where a function calls itself to solve a problem.

A pointer is a variable that stores the memory address of another variable.

6.3.1 Operations on Pointers

Pointers support the following arithmetic operations:

  • Addition: ptr + n
  • Subtraction: ptr - n
  • Increment: ptr++ or ++ptr
  • Decrement: ptr-- or --ptr

Note: Pointer arithmetic is scaled based on the data type size.

    printf("Address: %p, Value: %d\n", p, *p);      // p + 1

    printf("Address: %p, Value: %d\n", p, *p);      // p + 2

7.4.1 What is a Union?

A union is a user-defined data type where all members share the same memory location.

Key Difference from Structure:

  • Structure: Each member has separate memory
  • Union: All members share same memory (size = largest member)

7.4.2 Union Declaration and Usage

Syntax:

C
union union_name {
    datatype member1;
    datatype member2;
    ...
};

Example:

C

    int i;
    float f;
    char str[20];
};

int main() {
    union Data data;
    
    printf("Size of union: %zu bytes\n", sizeof(union Data));
    // Size = 20 (size of largest member - str array)
    
    data.i = 10;
    printf("data.i = %d\n", data.i);
    
    data.f = 3.14;
    printf("data.f = %.2f\n", data.f);

    printf("data.str = %s\n", data.str);
    
    return 0;
}

Important Note: In a union, only one member can hold a value at a time because they share memory.

9.7.1 Creating a Linked List

#include <stdio.h>
#include <stdlib.h>

struct Node {
    int data;
    struct Node *next;
};

struct Node *head = NULL;

// Insert at beginning
void insertAtBeginning(int value) {
    struct Node *newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->data = value;
    newNode->next = head;
    head = newNode;
}

// Insert at end
void insertAtEnd(int value) {
    struct Node *newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->data = value;
    newNode->next = NULL;
    
    if (head == NULL) {
        head = newNode;
    } else {
        struct Node *temp = head;
        while (temp->next != NULL) {
            temp = temp->next;
        }
        temp->next = newNode;
    }
}

// Display list
void display() {
    struct Node *temp = head;
    printf("Linked List: ");
    while (temp != NULL) {
        printf("%d -> ", temp->data);
        temp = temp->next;
    }
    printf("NULL\n");
}

// Free entire list
void freeList() {
    struct Node *temp;
    while (head != NULL) {
        temp = head;
        head = head->next;
        free(temp);
    }
}

int main() {
    insertAtBeginning(30);
    insertAtBeginning(20);
    insertAtBeginning(10);
    insertAtEnd(40);

What are runtime errors? How do you debug them? Explain with example.

Answer

Runtime Errors occur during program execution, after successful compilation.

Common Runtime Errors:

  1. Division by Zero
  2. Array Out of Bounds
  3. Null Pointer Dereference
  4. Stack Overflow

Example 1: Division by Zero

C
#include <stdio.h>

int main() {
    int a, b;
    
    printf("Enter two numbers: ");
    scanf("%d %d", &a, &b);
    
    // Debugging: Check before division
    if (b == 0) {
        printf("Error: Cannot divide by zero!\n");
    } else {
        printf("Division result: %d\n", a / b);
    }
    
    return 0;
}

Example 2: Array Bounds

C
#include <stdio.h>

int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    int index;
    
    printf("Enter index (0-4): ");
    scanf("%d", &index);
    
    // Debugging: Bounds checking
    if (index < 0 || index >= 5) {
        printf("Error: Index out of bounds!\n");
    } else {
        printf("Element at index %d: %d\n", index, arr[index]);
    }
    
    return 0;
}

Debugging Techniques:

  1. Add print statements to trace execution
  2. Use debugger (gdb) with breakpoints
  3. Enable compiler warnings
  4. Use static analysis tools
